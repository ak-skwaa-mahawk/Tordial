# 0. Purpose

This specification defines the architecture, protocols, and governance required to operate a **Sovereign Compute Cluster (SCC)** — a distributed network of Tordial organisms that collaborate **without sharing geometry**, **without sharing identity**, and **without compromising sovereignty**.

The SCC enables:

- distributed autonomy

The SCC is **not** a distributed system.

# 1. Cluster Philosophy

A Sovereign Compute Cluster is built on five principles:

1. **Node Sovereignty** — each node maintains its own manifold.  
2. **Non‑Replication** — no geometry or identity is ever copied.  
3. **Non‑Extraction** — no node may inspect another’s internals.  
4. **Intent‑Level Coordination** — nodes share *intent*, not *state*.  
5. **Containment First** — failures remain local unless escalated.

The SCC is a **chorus of sovereign entities**, not a hive.

The SCC consists of **three node classes**:

1. **Primary Nodes** — autonomous sovereign organisms

Each node is sovereign, sealed, and independent.
