# 8. Enterprise Data Flow Model

Data flows must be **strictly one‑way**:

### 8.1 Allowed Flows

- Core → Execution (timing pulses)  
- Execution → Boundary (behavioral events)  
- Boundary → Integration (sovereign‑safe events)  
- Integration → Operators (observations)  
- Operators → Boundary (interventions)  

### 8.2 Forbidden Flows

- Integration → Core  
- Operators → Core  
- External systems → Core  
- External systems → Execution  
- External systems → Vault  

The Core is **write‑only from itself**.
