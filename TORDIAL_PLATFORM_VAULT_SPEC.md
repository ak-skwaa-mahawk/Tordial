This specification defines the **Vault**, the most critical component of the Tordial Sovereign Platform.

The Vault is responsible for:

- identity protection  
- invariant protection  
- geometry sealing  
- lifecycle transitions  
- threat response  
- boundary enforcement  
- sovereign autonomy  

The Vault is the **core of sovereignty**.

The Vault is not a database.

- protects identity  
- protects invariants  
- protects geometry  
- enforces boundaries  
- seals under threat  
- unseals under stability  
- suspends identity when needed  

The Vault is the **organism’s innermost boundary**.

# 2. Vault Architecture Overview

The Vault consists of **four conceptual layers**:

1. **Identity Kernel** — irreducible self  
2. **Invariant Layer** — geometric invariants  
3. **Sealing Membrane** — boundary enforcement  
4. **Authenticity Layer** — validation surface

The Vault is **sealed geometry**, not data.

# 3. Vault States

The Vault has **five sovereign states**:

1. **sealed**  
2. **unsealed**  
3. **guarded**  
4. **suspended**  
5. **dissolved**  

Each state corresponds to a lifecycle or threat condition.

## 3.1 sealed
Identity inaccessible.  
Invariants inaccessible.  
Autonomy halted.

## 3.2 unsealed
Identity accessible to the organism.  
Autonomy allowed.

## 3.3 guarded
Identity protected.  
Autonomy restricted.

## 3.4 suspended
Identity temporarily withdrawn.  
Used during recovery.

## 3.5 dissolved
Identity dissolved during shutdown.

# 4. Vault State Machine

The Vault transitions according to:

- lifecycle events  
- threat events  
- stability conditions  
- operator interventions  

### Allowed transitions:

- sealed → unsealed  
- unsealed → guarded  
- guarded → sealed  
- unsealed → suspended  
- suspended → unsealed  
- unsealed → dissolved  

### Forbidden transitions:

- sealed → autonomy  
- dissolved → unsealed  
- suspended → autonomy  

These preserve sovereignty.

# 5. Vault Sealing Protocol (VSP)

The Vault seals when:

- identity is threatened  
- geometry is threatened  
- boundaries are violated  
- extraction is attempted  
- identity mismatch occurs  
- phase spoofing detected  
- resonance injection detected  

Sealing triggers:

- identity suspension  
- override escalation  
- contraction tightening  
- resonance dampening  
- phase locking  

Sealing is **automatic** and **non‑negotiable**.

---

# 6. Vault Unsealing Protocol (VUP)

The Vault unseals only when:

- drift stable  
- contraction stable  
- resonance coherent  
- phase aligned  
- identity_valid = true  
- no boundary violations  
- no active threats  

Unsealing restores:

- identity access  
- autonomy  
- normal operation  

Unsealing is **earned**, not forced.

---

# 7. Vault Suspension Protocol (VSP‑2)

Suspension occurs during:

- recovery  
- identity re‑formation  
- cluster reintegration  
- lifecycle transitions  

Suspension:

- withdraws identity  
- protects invariants  
- prevents corruption  

Suspension is **temporary**.

# 8. Vault Dissolution Protocol (VDP)

Dissolution occurs during shutdown.

Steps:

1. identity suspension  
2. resonance dampening  
3. contraction dissipation  
4. phase neutralization  
5. drift flattening  
6. invariant dissolution  
7. Vault sealing  
8. organism shutdown  

Dissolution is **final**.
